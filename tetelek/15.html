1. Az SQL adatbázisnyelv

SQL alapfogalmak

További részletek

Mi az SQL? Az SQL (Structured Query Language) a relációs adatbázis-kezelés szabványos nyelve, amely lehetővé teszi az adatok létrehozását, módosítását, lekérdezését és törlését.

SQL jellemzői:

  • Nem algoritmikus nyelv
  • Algoritmikus nyelvekbe beépíthető (beágyazott SQL)
  • Számos típusa és bővítése létezik (pl. PostgreSQL, PLSQL)
  • Nélkülözhetetlen a modern informatikában és az adatbázisokkal való munkában

SQL fő részei

További részletek

Az SQL utasításai két fő csoportba sorolhatók:

1. DDL (Data Definition Language):

  • Adatstruktúra definiáló utasítások
  • Az adatbázis szerkezetének meghatározása
  • Főbb DDL parancsok:
    • CREATE: Új adatbázis objektum létrehozása
    • ALTER: Meglévő objektum módosítása
    • DROP: Adatbázis objektum törlése

2. DML (Data Manipulation Language):

  • Adatokon műveletet végző utasítások
  • Az adatok kezelése és manipulálása
  • Főbb DML parancsok:
    • INSERT: Új adatok beszúrása
    • UPDATE: Meglévő adatok módosítása
    • DELETE: Adatok törlése
    • SELECT: Adatok lekérdezése

Összefoglalás: A DDL parancsokkal határozzuk meg az adatbázis szerkezetét, míg a DML parancsokkal az adatokat kezeljük. Együtt biztosítják az adatbázisok struktúrájának és tartalmának teljes körű kezelését.

2. Relációsémák definiálása, megszorítások típusai és létrehozásuk

Relációsémák alapfogalmai

További részletek

Mi a relációséma? A relációsémák határozzák meg a táblák szerkezetét és az adatok közötti kapcsolatokat az adatbázisokban.

Hogyan hozunk létre relációsémát? Relációséma létrehozására a CREATE TABLE utasítást használjuk, amely egyben egy üres táblát is létrehoz a sémához.

Adattípusok

További részletek

Leggyakoribb adattípusok:

  • CHAR(n): n hosszúságú karaktersorozat
  • VARCHAR(n): legfeljebb n hosszúságú karaktersorozat
  • INTEGER: egész szám (röviden INT)
  • REAL: valós (lebegőpontos) szám, másnéven FLOAT
  • DECIMAL(n[,d]): n jegyű decimális szám, ebből d tizedesjegy
  • DATE: dátum (év, hó, nap)
  • TIME: idő (óra, perc, másodperc)

Megszorítások típusai

További részletek

Mire szolgálnak a megszorítások? A megszorítások segítségével biztosíthatjuk az adatok érvényességét és konzisztenciáját.

Oszlopfeltételek:

  • PRIMARY KEY: elsődleges kulcs (csak egy lehet)
  • UNIQUE: kulcs (több is lehet)
  • REFERENCES tábla(oszlop): külső kulcs
  • NOT NULL: NULL érték tiltása

Táblafeltételek:

  • PRIMARY KEY (oszloplista): elsődleges kulcs
  • UNIQUE (oszloplista): kulcs
  • FOREIGN KEY (oszloplista) REFERENCES tábla(oszloplista): külső kulcs

ON-feltételek: Szabályozzák a rendszer viselkedését külső kulcsok esetén:

  • ON UPDATE/DELETE CASCADE: módosítás továbbgyűrűzése
  • ON UPDATE/DELETE SET NULL: NULL érték beállítása módosításkor

Séma módosítása és törlése

További részletek

Hogyan módosítunk sémát? Relációséma módosítására az ALTER TABLE utasítást használjuk.

Hogyan törlünk sémát? Séma és a hozzá tartozó adattábla törlésére a DROP TABLE utasítás szolgál.

Új séma létrehozása

További részletek

Hogyan hozunk létre új sémát? Új séma létrehozásához a CREATE TABLE utasítást használjuk.

Alapvető szintaxis:

CREATE TABLE tabla_neve (
    oszlop1 adattípus1 [megszorítások],
    oszlop2 adattípus2 [megszorítások],
    ...
    [tábla szintű megszorítások]
);

Példa:

CREATE TABLE Dolgozok (
    id INT PRIMARY KEY,
    nev VARCHAR(50) NOT NULL,
    fizetes DECIMAL(10, 2),
    osztaly_id INT,
    FOREIGN KEY (osztaly_id) REFERENCES Osztalyok(id)
);

Mire kell figyelni? A tábla és oszlop nevek egyediek legyenek, és válasszuk meg megfelelően az adattípusokat és megszorításokat.

3. Adatmanipulációs lehetőségek és lekérdezések

Adatmanipulációs utasítások

További részletek

INSERT INTO: Értékek beszúrása a megadott tábla megadott oszlopaiba.

INSERT INTO táblanév [(oszloplista)] VALUES (értéklista)

Megjegyzés: Ha oszloplista nem szerepel, akkor valamennyi oszlop értéket kap a CREATE TABLE-ben megadott sorrendben.

UPDATE: Értékek aktualizálása.

UPDATE táblanév SET oszlop = kifejezés, ..., oszlop = kifejezés [ WHERE feltétel ]

Megjegyzés: Az értékadás minden olyan soron végrehajtódik, amely eleget tesz a WHERE feltételnek. Ha WHERE feltétel nem szerepel, akkor az értékadás az összes sorra megtörténik.

DELETE FROM: Értékek törlése.

DELETE FROM táblanév [ WHERE feltétel ]

Megjegyzés: Hatására azok a sorok törlődnek, amelyek eleget tesznek a WHERE feltételnek. Ha a WHERE feltételt elhagyjuk, akkor az összes sor törlődik.

Lekérdezések (SELECT)

További részletek

Mi a SELECT utasítás? A SELECT utasítás szolgál lekérdezésre, amely egy vagy több adattáblából egy eredménytáblát állít elő.

SELECT utasítás alapvető formája:

       SELECT [DISTINCT] oszloplista 
        FROM táblanevek 
        [WHERE feltétel];
  • DISTINCT: Csak különböző értékek lekérése
  • *: Az összes oszlop kiválasztása
  • Kifejezések: A SELECT után nem csak oszlopnevek, hanem tetszőleges kifejezések is állhatnak (pl. COUNT, *, +)
  • Alias nevek: AS kulcsszóval adhatunk alias neveket (pl. SELECT table AS t)

Függvények és összesítő függvények:

  • Egyszerű függvények: ABS, TO_CHAR, TO_DATE, stb.
  • Összesítő függvények: AVG, SUM, MAX, MIN, COUNT

Példa: SELECT AVG(fizetés) FROM Dolgozó

Csoportosítás (GROUP BY, HAVING):

Ha a tábla sorait csoportonként szeretnénk összesíteni, akkor a SELECT utasítás a GROUP BY oszloplista alparanccsal bővítendő.

Példa: SELECT osztkód, AVG(fizetés) FROM Dolgozó GROUP BY osztkód;

Csoportosítási szabály: A SELECT után összesítő függvényen kívül csak olyan oszlopnév tüntethető fel, amely a GROUP BY-ban is szerepel.

HAVING: A GROUP BY által képezett csoportok közül válogathatunk.

Példa:

SELECT osztkód, AVG(fizetés)
FROM Dolgozó
GROUP BY osztkód HAVING AVG(fizetés) > 180000

Rendezés (ORDER BY):

Az eredménytábla rendezésére szolgál. A SELECT utasítás végére helyezhető.

Szintaxis: ORDER BY oszlopnév [DESC], ..., oszlopnév [DESC]

SELECT utasítás általános alakja:

SELECT [DISTINCT] oszloplista
FROM táblanévlista
[JOIN táblanév ON t.oszlop = t1.oszlop]
[WHERE feltétel]
[GROUP BY oszloplista
[HAVING feltétel] ]
[ORDER BY oszloplista];

Táblaösszekapcsolások:

  • INNER JOIN: Csak azokat a sorokat adja vissza, ahol mindkét táblában van egyezés. 
    SELECT *
FROM tabla1
INNER JOIN tabla2 ON tabla1.id = tabla2.tabla1_id;
  • LEFT JOIN: Az első tábla minden sorát visszaadja, még ha nincs is egyezés a második táblában. 
    SELECT *
FROM tabla1
LEFT JOIN tabla2 ON tabla1.id = tabla2.tabla1_id;
  • RIGHT JOIN: A második tábla minden sorát visszaadja, még ha nincs is egyezés az első táblában. 
    SELECT *
FROM tabla1
RIGHT JOIN tabla2 ON tabla1.id = tabla2.tabla1_id;
  • FULL OUTER JOIN: Mindkét tábla összes sorát visszaadja, függetlenül attól, hogy van-e egyezés. 
    SELECT *
FROM tabla1
FULL OUTER JOIN tabla2 ON tabla1.id = tabla2.tabla1_id;

Alkérdések: Ha egy SELECT utasítás WHERE vagy HAVING feltételében olyan logikai kifejezés szerepel, amely SELECT utasítást tartalmaz, ezt alkérdésnek vagy belső SELECT-nek nevezik.

Példa:

SELECT név, fizetés FROM Dolgozó WHERE fizetés < (SELECT AVG(fizetés) FROM dolgozó)

Nézettáblák

További részletek

Mi a nézettábla? A nézettábla (view) egy virtuális tábla, amely nem tárol adatokat, hanem egy transzformációs formula segítségével állítja elő az eredményt.

Hogyan hozunk létre nézettáblát?

CREATE VIEW táblanév [(oszloplista)] AS alkérdés;

Nézettáblák alkalmazási lehetőségei:

  • Származtatott adattáblák létrehozása, amelyek a törzsadatok módosításakor automatikusan módosulnak (pl. összegzőtáblák)
  • Bizonyos adatok elrejtése egyes felhasználók elől (adatbiztonság vagy egyszerűsítés céljából)

Beágyazott SQL és API-k

További részletek

Mi a beágyazott SQL? Az SQL utasításokat egy hagyományos algoritmikus programnyelv (C, Java, stb.) utasításaival keverten használjuk. Ezt nevezzük beágyazott (embedded) SQL-nek.

ODBC (Open DataBase Connection):

  • Általános API különböző adatbázis-kezelő rendszerekhez való csatlakozásra
  • Platformfüggetlen és nyelvfüggetlen
  • Főbb lépések: Connect - Prepare - Execute
  • Példa függvények:
    • SQLDriverConnect(dbc, ...);
    • SQLPrepare(stmt, "SELECT cikkszám, egységár FROM Raktár", SQL_NTS);
    • SQLExecute(stmt);

JDBC (Java DataBase Connection):

  • Java specifikus API adatbázisokhoz
  • Az ODBC-hez hasonló, de objektumorientált jellegű
  • Példa metódusok:
    • DriverManager.getConnection(url, user, password)
    • CreateStatement(SQL lekérdezés vagy utasítás létrehozása)
    • executeQuery(SQL lekérdezés vagy utasítás végrehajtása)

Előkészített SQL lekérdezések előnyei:

  • Biztonság: Csökkenti az SQL Injection kockázatát
  • Teljesítmény: Gyorsabb futás optimalizált és cache-elt végrehajtási tervekkel
  • Hatékonyság: Erőforrás-takarékosság ismételt végrehajtásnál
  • Kód tisztasága: Olvashatóbb és karbantarthatóbb kód, jobb hibakezelés

4. Összefoglalás

Az SQL adatbázisnyelv: DDL és DML

További részletek

1. Adatdefiníciós nyelv (DDL)

  • A struktúra meghatározása: táblák, indexek stb. létrehozása
  • Alapvető parancsok: CREATE TABLE, ALTER TABLE, DROP TABLE stb.

2. Adatmanipulációs nyelv (DML)

  • Adatok manipulálása: beszúrás, frissítés, törlés
  • Alapvető parancsok: INSERT, UPDATE, DELETE

Relációsémák és megszorítások

További részletek

1. Relációsémák

  • Táblák létrehozása és attribútumok meghatározása
  • Kulcsok, indexek és megszorítások hozzáadása

2. Megszorítások típusai

  • PRIMARY KEY, UNIQUE, FOREIGN KEY, NOT NULL stb.
  • Adatintegritás és konzisztencia biztosítása

Adatmanipulációs lehetőségek és lekérdezések

További részletek

1. Adatmanipulációs lehetőségek

  • Adatok beszúrása, frissítése és törlése táblákból
  • Alapvető parancsok: INSERT INTO, UPDATE, DELETE

2. Lekérdezések

  • Adatok lekérdezése táblákból
  • Filterezés, aggregálás, csoportosítás és rendezés
  • Alapvető parancs: SELECT

API-k: ODBC és JDBC

További részletek

1. ODBC (Open DataBase Connection)

  • Általános API különböző adatbázis-kezelő rendszerekhez való csatlakozáshoz
  • Platform- és nyelvfüggetlen kommunikáció
  • Connect - Prepare - Execute

2. JDBC (Java DataBase Connection)

  • Java specifikus API adatbázisokhoz való kapcsolódáshoz és műveletek végrehajtásához

Az előkészített SQL lekérdezések szerepe

További részletek

Előnyök:

  • Biztonság: Az SQL Injection kockázatának csökkentése
  • Teljesítmény: Gyorsabb futás és optimalizált végrehajtási tervek
  • Hatékonyság: Ismételt végrehajtás esetén erőforrások takarékossága
  • Kód minősége: Olvashatóbb és karbantarthatóbb kód, szintaktikai hibák előzetes felderítése

Záró gondolatok

További részletek

Az SQL alapok, a DDL és a DML felölelik az adatbázisok szerkezetét és manipulációját. A relációsémák létrehozása és a megszorítások bevezetése biztosítja az adatok integritását és konzisztenciáját. Az adatmanipuláció során az INSERT, UPDATE és DELETE utasításokkal kezeljük az adatokat, míg a lekérdezések segítségével részletesen vizsgálhatjuk azokat. Az API-k, mint az ODBC és JDBC, teljessé teszik az adatbázisok kezelését a különböző programozási nyelvekben, és az előkészített utasítások használata javítja a biztonságot és a teljesítményt.

Az SQL helyes használata elengedhetetlen a hatékony adatkezeléshez és az adatbázisokkal való munkához.
