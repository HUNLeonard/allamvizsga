1. Egyszerű adattípusok

Egyszerű adattípusok áttekintése

További részletek

Mi az adattípus? Az adattípus olyan egysége a programnak, amely annak értékhalmaza és az értékhalmazán végezhető műveletek alapján pontosan meghatározható.

Milyen csoportokra oszthatjuk az adattípusokat? Az adattípusokat egyszerű (elemi) és összetett típusokra oszthatjuk.

Mik az elemi adattípusok? Az elemi adattípusok olyan típusok, amelyeket nem lehet önmagukban értelmes részekre bontani. Ide tartoznak az egész, valós, logikai és karakter típusok, valamint a felsorolás (enum).

Melyek az egyszerű adattípusok C-ben?

  • Egész számok (int, short, long, long long)
  • Valós számok (float, double, long double)
  • Karakter (char)
  • Logikai érték (bool - C99 szabványtól)
  • Felsorolás (enum)

Egész számok

További részletek

Milyen egész típusok vannak? A C nyelvben többféle egész típus létezik, különböző méretekben és előjellel vagy anélkül.

Hány bites egy int C-ben? Általában 32 bit, de ez platformfüggő lehet. Régebbi rendszereken 16 bit is lehet.

Mi a signed és unsigned int értéktartománya?

  • signed int (32 bit): [-2,147,483,648 to 2,147,483,647]
  • unsigned int (32 bit): [0 to 4,294,967,295]

Típusok és jellemzőik:

  • short: 16 bit, [-32,767, 32,767] vagy [0, 65,535] (unsigned)
  • int: 16 vagy 32 bit (platformfüggő), [-32,767, 32,767] vagy [0, 65,535] (16 bit esetén)
  • long: 32 bit, [-2 milliárd, 2 milliárd] vagy [0, 4 milliárd] (unsigned)
  • long long: 64 bit, [-sok, sok] vagy [0, rengeteg] (unsigned)

Milyen műveletek végezhetők rajtuk?

  • Aritmetikai: +, -, *, /, % (maradékos osztás)
  • Összehasonlító: ==, !=, <, >, <=, >=
  • Bitműveletek: &, |, ^, ~, <<, >>

Mi a túlcsordulás? Túl- és alulcsordulás előfordulhat, ha a kereten kívül esik a reprezentált érték. Ez hibás eredményekhez vezethet.

Hogyan érjük el a határértékeket? A limits.h header fájl tartalmazza az egész típusok határértékeit, például INT_MIN, INT_MAX.

Mi az a kettes komplemens? A kettes komplemens a negatív számok ábrázolási módja. A pozitív szám bitjeit invertáljuk, majd hozzáadunk 1-et.

Hogyan néz ki a -1 kettes komplemensben? 32 biten ábrázolva: 11111111 11111111 11111111 11111111

Valós számok

További részletek

Mik a valós számok? A valós számok lebegőpontos ábrázolással tárolódnak a számítógép memóriájában.

Hogyan tárolódnak a valós számok a memóriában? A valós számok lebegőpontos ábrázolással tárolódnak, amely három részből áll: előjel, mantissza és kitevő.

Mi a float ábrázolásának felbontása?

  • 1 bit előjel
  • 8 bit kitevő
  • 23 bit mantissza

Típusok és jellemzőik:

  • float: 32 bit (1 előjelbit, 8 bit kitevő, 23 bit mantissza)
  • double: 64 bit (1 előjelbit, 11 bit kitevő, 52 bit mantissza)
  • long double: 128 bit (1 előjelbit, 15 bit kitevő, 112 bit mantissza)

Milyen műveletek végezhetők rajtuk?

  • Aritmetikai: +, -, *, /
  • Összehasonlító: ==, !=, <, >, <=, >=
  • Matematikai függvények: sin, cos, log, exp (math.h)
Összetett adattípusok

Hogyan ábrázolunk egy nagyon nagy és egy nagyon kis számot? A kitevő segítségével tudjuk kezelni a nagyon nagy és nagyon kis számokat. A nagy számoknál pozitív, a kis számoknál negatív kitevőt használunk.

double nagy_szam = 1.0e12;  // 1 billió
double kis_szam = 1.0e-9;   // 1 milliárdod

printf("%f\n", nagy_szam); // for floating-point format
printf("%e\n", nagy_szam); // for scientific notation format

Mi a pontosság kérdése? A lebegőpontos számok korlátozott pontosságúak, kerekítési hibák előfordulhatnak. Összehasonlításnál különös óvatosság szükséges.

Logikai típus

További részletek

Mi a logikai típus? A logikai típus két lehetséges értéket tárolhat: igaz vagy hamis.

Jellemzők:

  • Típus: bool
  • Értékkészlet: true, false
  • Memóriaigény: 8 bit (1 byte)

Milyen műveletek végezhetők rajta?

  • Logikai NOT: !
  • Logikai AND: &&
  • Logikai OR: ||
  • Összehasonlítás: ==, !=

Mikor került be a C nyelvbe? A bool típus a C99 szabvánnyal került be a nyelvbe. Használatához szükséges a stdbool.h header fájl.

Karakter típus

További részletek

Mi a karakter típus? A karakter típus egy karaktert (betű, szám, írásjel) ábrázol, valójában egy kis egész számként tárolva.

Jellemzők:

  • Típus: char, signed char, unsigned char
  • Értékkészlet: [-128, 127] vagy [0, 255] (előjeles vagy előjel nélküli)
  • Memóriaigény: 8 bit (1 byte)

Milyen műveletek végezhetők rajta?

  • Aritmetikai műveletek (ASCII kód alapján)
  • Összehasonlítás
  • Bitműveletek

Hogyan ábrázoljuk a karaktereket? A karakterek ASCII kódként tárolódnak. Értékadás történhet karakterkóddal számként vagy aposztrófok közé írt karakterrel.

Felsorolás (enum)

További részletek

Mi az enum? Az enum egy felhasználó által definiált adattípus, amely neveket rendel egész konstansokhoz.

Jellemzők:

  • Alapértelmezetten az értékek 0-tól kezdődnek és növekednek
  • Explicit értékek is megadhatók
  • Minden enum konstans egyedi kell legyen a saját hatókörében

Mire használjuk? Az enum-ot gyakran használják a kód olvashatóságának javítására és állapotok reprezentálására.

Példa:

enum State {Working = 1, Failed = 0};

Lehet-e azonos értékű két enum név? Igen, két enumerációs név azonos értékkel rendelkezhet. Például: enum {Failed = 0, Freezed = 0};

2. Összetett adattípusok

Összetett adattípusok áttekintése

További részletek

Mik az összetett adattípusok? Az összetett adattípusok több adatelemet tartalmaznak, amelyek lehetnek azonos vagy különböző típusúak.

Melyek a C nyelvben elérhető összetett adattípusok?

  • Tömb (Array)
  • Struktúra (struct)
  • Unió (union)
  • Pointer

Pointer

További részletek

Mi a pointer? A pointer egy változó, amely egy másik változó memóriacímét tárolja értékként.

Hogyan hozzuk létre?

  • A * operátorral definiáljuk, például: int *p;
  • A p egy int-re mutató pointer, míg a *p a mutatott int érték

Milyen műveletek végezhetők rajtuk?

  • NULL: amikor a pointerhez nem tartozik dinamikus változó
  • Létesít (malloc): új, dinamikus változó létesítése
  • Törlés (free): dinamikus változó törlése, memória felszabadítása
  • Dereferencia (*): mutatott érték elérése
  • Pointer aritmetika: Pointerek növelése/csökkentése
  • (Nem)Egyenlő (==, !=): két pointer értékének összehasonlítása
  • Cím lekérése (&): változó címének lekérése

Miért fontos a pointer típusa? A pointer típusa meghatározza, hogyan értelmezzük a mutatott memóriaterületet és mennyi memóriát foglalunk/szabadítunk fel műveletek során.

Mekkora memóriát foglal egy pointer tömb? Egy pointer tömb mérete függ a rendszertől (32 vagy 64 bit), és a tömb elemszámától. Például egy 5 elemű int* tömb 64 bites rendszeren: 5 * 8 byte = 40 byte.

Mire kell figyelni használatuk során? A pointerek használata körültekintést igényel a memóriakezelés és a hibák elkerülése érdekében. Fontos a megfelelő inicializálás és a NULL-ellenőrzés.

Hogyan foglalunk dinamikusan memóriát? A malloc() függvénnyel, például: int *p = (int *)malloc(sizeof(int));. Használat után a free() függvénnyel kell felszabadítani a memóriát.

Struktúra (struct) és Unió (union)

További részletek

Mi a különbség a struct és a union között?

Struct:

  • Különböző típusú adatokat tárol együtt
  • Minden tag saját memóriaterületet kap
  • Mérete általában a tagok méretének összege (plusz esetleges padding)

Union:

  • Különböző típusú adatokat tárol ugyanazon a memóriaterületen
  • Mérete a legnagyobb tag méretével egyezik meg
  • Egyszerre csak egy tagja használható biztonságosan

Hogyan vannak megvalósítva? A struct tagjai egymás után helyezkednek el a memóriában, míg a union tagjai ugyanazt a memóriaterületet használják, csak különböző módon értelmezve azt.

Tömb

További részletek

Mi a tömb? A tömb azonos típusú elemek fix méretű, folytonos gyűjteménye a memóriában.

Jellemzők:

  • Rögzített méretű
  • Elemei azonos típusúak
  • Indexelés 0-tól kezdődik

Milyen műveletek végezhetők rajta?

  • Kiolvas: Adott indexű elem értékének lekérése
  • Módosít: Adott indexű elem értékének módosítása
  • Értékadás: Teljes tömb másolása (C-ben korlátozottan)

Mire kell figyelni használatuk során? A tömbindexelés során nincs automatikus határellenőrzés, így könnyen túlindexelhetünk, ami hibákhoz vezethet.

Tömbök és Pointerek összehasonlítása

További részletek

Mi a fő különbség a tömbök és pointerek között?

Tömbök:

  • Fix méretűek, fordítási időben meghatározva
  • Nem lehet újradeklarálni vagy átméretezni
  • Automatikusan lefoglalt memóriaterületen helyezkednek el

Pointerek:

  • Dinamikusan foglalható és felszabadítható memóriára mutatnak
  • Méretük változtatható futási időben
  • Null értéket is felvehetnek

Hogyan kapcsolódnak egymáshoz? C-ben a tömbnév gyakran automatikusan pointerre konvertálódik, ami a tömb első elemére mutat. Ez lehetővé teszi a tömbök pointer-szerű használatát.

Rekord (struct)

További részletek

Mi a struct? A struct egy felhasználó által definiált adattípus, amely különböző típusú adatelemeket (mezőket) foglal magában.

Jellemzők:

  • Különböző típusú adatelemeket tartalmazhat
  • Az elemek egymás után helyezkednek el a memóriában
  • A struct mérete általában nagyobb vagy egyenlő, mint az elemek méretének összege

Milyen műveletek végezhetők rajta?

  • Kiolvas (→): visszaadja az adott komponens értékét
  • Módosít (↔): módosítja az adott komponens értékét
  • Értékadás (←): értéket ad az adott komponens értékének

Hogyan érjük el a struct mezőit? A mezőket a pont (.) operátorral érjük el, például: szemely.nev. Pointer esetén a nyíl (->) operátort használjuk: szemely_ptr->nev.

Példa:

struct Szemely {
    char nev[50];
    int kor;
    float magassag;
};

Unió (union)

További részletek

Mi az unió? Az unió egy olyan adatszerkezet, amely több adattag közül egyszerre csak egyet tárol ugyanazon a memóriaterületen.

Jellemzők:

  • Több adattag, de csak egy aktív egyszerre
  • Mérete a legnagyobb adattag méretével egyezik meg
  • Memóriatakarékos, ha az adattagok kölcsönösen kizárják egymást

Milyen műveletek végezhetők rajta?

  • Kiolvas: Aktuális adattag értékének lekérése
  • Módosít: Adattag értékének módosítása
  • Értékadás: Új érték beállítása (aktív adattag változik)

Mire kell figyelni használatuk során? Az unió használata óvatosságot igényel, mert nem követi nyomon, melyik adattag aktív éppen. A programozónak kell gondoskodnia erről.

Példa:

union Adat {
    int i;
    float f;
    char str[20];
};

3. Összefoglalás és Vizsgakérdések

Gyakori Vizsgakérdések

További részletek

  • Mi az adattípus definíciója?
  • Hogyan csoportosítjuk az adattípusokat C-ben?
  • Milyen méretű egy int C-ben? Mi a signed és unsigned értéktartománya?
  • Hogyan néz ki a -1 kettes komplemens ábrázolásban?
  • Hogyan tárolódnak a valós számok a memóriában? Mi a float ábrázolásának felbontása?
  • Mi a különbség a struct és a union között? Hogyan vannak megvalósítva?
  • Milyen műveletek végezhetők pointerekkel? Miért fontos a pointer típusa?
  • Mi a fő különbség a tömbök és pointerek között? Hogyan kapcsolódnak egymáshoz?

Egyszerű adattípusok összefoglalása

További részletek

  • Egész számok: Különböző méretű és előjelű változatok, pontos aritmetika
  • Valós számok: Lebegőpontos ábrázolás, korlátozott pontosság
  • Logikai típus: Igaz/hamis értékek, logikai műveletek
  • Karakter: ASCII karakterek tárolása, valójában kis egész szám
  • Felsorolás (enum): Névvel ellátott konstansok definiálása

Összetett adattípusok összefoglalása

További részletek

  • Pointer: Memóriacímek tárolása, dinamikus memóriakezelés
  • Tömb: Azonos típusú elemek fix méretű gyűjteménye
  • Rekord (struct): Különböző típusú adatelemek csoportosítása
  • Unió (union): Több adattípus tárolása ugyanazon memóriaterületen

Záró gondolatok

További részletek

Az adattípusok megfelelő megválasztása és használata kulcsfontosságú a hatékony és megbízható programok írásához. Az egyszerű adattípusok biztosítják az alapvető építőelemeket, míg az összetett adattípusok lehetővé teszik komplex adatstruktúrák létrehozását. A C nyelv erőssége, hogy alacsony szintű memóriakezelést tesz lehetővé, ami nagy rugalmasságot biztosít, de egyben felelősséget is ró a programozóra a helyes és biztonságos használat tekintetében.
